# Watchtower

A personal health tracker for logging meals, blood glucose readings, and medications — with AI-powered insights built in.

**Live app:** [watchtower.aniketraj.me](https://watchtower.aniketraj.me)

---

## What it does

Watchtower gives you a private, self-hosted space to track the health data that matters most to you. Log what you eat, your blood glucose readings, and whether you've taken your medications. At the end of each day, Claude generates a digest that looks at patterns in your data. Throughout the day, a live Intel card gives you context-aware recommendations based on your recent meals and readings.

You can log entries the normal way — or just type a single sentence in plain English and let the AI parse and store it for you.

**Key features:**

- Log meals with meal type, description, and optional medication tracking
- Log blood glucose readings linked to specific meals
- Track medications (name, dose, frequency, schedule)
- Natural-language logging — type `"had oatmeal and OJ, bg was 145 after"` and it handles the rest
- Daily digest generated by Claude Sonnet each morning at 2:00 AM
- Live Intel recommendations cached per time-of-day bucket, refreshable on demand
- Full-day view for reviewing any past date
- Installable as a PWA

## Running locally

### Prerequisites

- Node.js ≥ 20
- An [Anthropic API key](https://console.anthropic.com/) — only required for AI features (natural-language logging, Intel, daily digest)

### Setup

```bash
git clone https://github.com/aniketrajfetch/watchtower.git
cd watchtower
npm install
cp .env.example .env
```

Edit `.env` and fill in the variables:

| Variable | Required | Purpose |
|---|---|---|
| `APP_PASSWORD` | Yes | Password you enter to sign in to the web app |
| `SESSION_SECRET` | Yes | Secret used to sign session cookies — use a long random string |
| `ANTHROPIC_API_KEY` | For AI features | Powers natural-language logging, Intel, and daily digests |
| `PORT` | No (default `3000`) | HTTP port to listen on |
| `SERVER` | No | Label that appears in the `/health` response |

### Start

```bash
# Development (auto-reload)
npm run dev

# Production
npm start
```

The app is served at `http://localhost:3000`.

### Authentication

Sign-in uses an **HttpOnly session cookie** — there are no client-visible API tokens. All `/api/*` routes (except login, logout, and session check) require a valid session.

**Scripting with curl:**

```bash
# Sign in and save the session cookie
curl -c cookies.txt -X POST http://localhost:3000/api/login \
  -H 'Content-Type: application/json' \
  -d '{"password":"YOUR_APP_PASSWORD"}'

# Use the session cookie in subsequent requests
curl -b cookies.txt http://localhost:3000/api/readings

# Force-refresh the live Intel recommendation
curl -b cookies.txt -X POST http://localhost:3000/api/intel/recommendation \
  -H 'Content-Type: application/json' \
  -d '{"refresh":true}'
```

### Deploying to Railway

Set `APP_PASSWORD`, `SESSION_SECRET`, and `ANTHROPIC_API_KEY` in Railway Variables. The `railway.toml` is already configured with the start command and a healthcheck at `GET /health`. Mount a **volume at `/data`** and set `DATABASE_PATH=/data/watchtower.db` so the SQLite database survives redeploys.

## API reference

All endpoints that return or accept data use `application/json`. Session-protected routes require a valid `wt_session` cookie obtained via `POST /api/login`.

**Timestamp fields** (`timestamp` on meals, readings, and natural-language log) are optional ISO-8601 strings. If provided, they must fall within the last 7 days and no more than 5 minutes in the future. Omit to use the server's current time.

---

### Health & Auth

#### `GET /health`

Public. Returns server status.

```json
{ "watchtower": "ok", "server": "production", "runtimeSec": 3600 }
```

---

#### `POST /api/login`

Public. Signs in and sets a session cookie.

**Body:**
```json
{ "password": "your-app-password" }
```

**Response `200`:**
```json
{ "ok": true }
```

Returns `401` on wrong password, `500` if env is misconfigured.

---

#### `POST /api/logout`

Public. Clears the session cookie.

**Response `200`:**
```json
{ "ok": true }
```

---

#### `GET /api/session`

Public. Check whether the current cookie is valid.

**Response `200`** (valid session): `{ "ok": true }`
**Response `401`** (no session or invalid): `{ "error": "Unauthorized" }`

---

### Meals

#### `POST /api/meals`

Log a meal. Requires session.

**Body:**

| Field | Type | Required | Notes |
|---|---|---|---|
| `meal_type` | string | Yes | e.g. `"breakfast"`, `"lunch"`, `"dinner"`, `"snack"` |
| `description` | string | Yes | Free-text description of the meal |
| `medication_taken` | boolean | No | If truthy and `meal_type` is breakfast/lunch/dinner, records a snapshot of active medications scheduled for that slot |
| `timestamp` | string (ISO-8601) | No | Defaults to now |
| `raw_input` | string | No | Original text if logged via natural language |

**Response `201`:** Full meal row.

---

#### `GET /api/meals`

List all meals, newest first. Requires session.

---

#### `GET /api/meals/today`

Today's meals. Requires session.

---

#### `DELETE /api/meals/:id`

Delete a meal by ID. Clears any readings that referenced this meal. Requires session.

**Response `200`:** `{ "deleted": true }`
**Response `404`:** Meal not found.

---

### Readings

#### `POST /api/readings`

Log a blood glucose reading. Requires session.

**Body:**

| Field | Type | Required | Notes |
|---|---|---|---|
| `reading_type` | string | Yes | e.g. `"fasting"`, `"post_meal"`, `"random"` |
| `bg_value` | number | Yes | Blood glucose value |
| `meal_ids` | number[] | No | Array of meal IDs to associate with this reading |
| `timestamp` | string (ISO-8601) | No | Defaults to now |
| `raw_input` | string | No | Original text if logged via natural language |

**Response `201`:** Reading row with `meal_ids` expanded to full meal objects:
```json
{
  "id": 12,
  "reading_type": "post_meal",
  "bg_value": 145,
  "timestamp": "2026-04-04T13:30:00.000Z",
  "meal_ids": [
    { "id": 8, "meal_type": "lunch", "timestamp": "...", "description": "grilled chicken salad" }
  ]
}
```

---

#### `GET /api/readings`

List all readings, newest first, with `meal_ids` expanded. Requires session.

---

#### `GET /api/readings/today`

Today's readings with `meal_ids` expanded. Requires session.

---

#### `DELETE /api/readings/:id`

Delete a reading by ID. Requires session.

**Response `200`:** `{ "deleted": true }`
**Response `404`:** Reading not found.

---

### Medications

#### `GET /api/medications`

List all medication rows. Requires session.

---

#### `PATCH /api/medications/:id`

Update a medication. Requires session.

**Body** (any combination of):

| Field | Type |
|---|---|
| `name` | string |
| `dose` | string |
| `frequency` | string |
| `schedule` | string |
| `is_default` | boolean |
| `is_active` | boolean |
| `notes` | string |

**Response `200`:** Updated medication row.
**Response `404`:** Medication not found.
**Response `400`:** No valid fields provided.

---

### Day View

#### `GET /api/day/:date`

All meals and readings for a specific date. Requires session.

`:date` must be `YYYY-MM-DD`.

**Response `200`:**
```json
{
  "meals": [...],
  "readings": [...]
}
```

Readings are returned with `meal_ids` expanded to full meal objects.

---

### Natural Language Log

#### `POST /api/log/natural`

Parse a free-text sentence and save the resulting meal, reading, or both. Requires session. Requires `ANTHROPIC_API_KEY`.

**Body:**

| Field | Type | Required | Notes |
|---|---|---|---|
| `text` | string | Yes | Free-text input, e.g. `"had oatmeal and OJ, bg was 145 after"` |
| `timestamp` | string (ISO-8601) | No | Applied to any saved entries |

**Response `201`:**
```json
{
  "signal": "received",
  "saved": {
    "meal": { ...meal row or null },
    "reading": { ...reading row or null }
  }
}
```

On parse failure or LLM error:
```json
{ "signal": "lost", "error": "..." }
```

---

### Intel

#### `POST /api/intel/recommendation`

Get a context-aware recommendation based on today's meals, readings, and the current time of day. Results are cached for ~30 minutes per time bucket. Requires session. Requires `ANTHROPIC_API_KEY`.

**Body** (all optional):

| Field | Type | Notes |
|---|---|---|
| `current_time` | string (`HH:MM`) | Overrides server time for bucket calculation |
| `refresh` | boolean | Bypass the cache and regenerate |

**Response `200`:**
```json
{
  "headline": "Good control this morning",
  "body": "...",
  "reasoning": "...",
  "type": "positive",
  "confidence": "high",
  "earned_treat": false,
  "treat_message": null,
  "data_days": 7,
  "generated_at": "2026-04-04T13:00:00.000Z"
}
```

---

#### `POST /api/intel/generate-digest`

Generate and store a daily digest for a given date. Skipped if there are no readings for that day. Runs automatically at 2:00 AM for the previous day; call this manually to regenerate or backfill. Requires session. Requires `ANTHROPIC_API_KEY`.

**Body** (optional):

| Field | Type | Notes |
|---|---|---|
| `date` | string (`YYYY-MM-DD`) | Defaults to yesterday |

**Response `200` (generated):** Full `daily_insights` row.
**Response `200` (skipped — no data):** `{ "skipped": true, "date": "2026-04-03" }`

---

#### `GET /api/intel/digests`

Retrieve recent daily digests. Requires session.

**Query params:**

| Param | Default | Notes |
|---|---|---|
| `days` | `14` | Number of digests to return (clamped to 1–30) |

**Response `200`:** Array of `daily_insights` rows, newest first.
